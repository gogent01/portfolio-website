  Which topics should be covered by a project architecture in the domain of major class structure? (Name 4 of them.)

- major classes and their responsibilities
- interclass communications
- relationships between classes: class hierarchies and subsystems
- other class designs considered and reasons for the final choice

Following 20/80 rule is enough: list 20% of classes responsible for 80% of the system's functionality. The rest can be specified at construction time.
