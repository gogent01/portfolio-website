question: |
  Which typical domains of program development should be covered by a solid project architecture? (Name 18)
publishedOn: '2024-09-02T12:00:00+0100'
id: QXs21g
---
- program organization
- major classes and subsystems
- business rules
- data design
- user interface design
- resource management
- security
- performance
- scalability
- interoperability (sharing of data or resources with other systems)
- internationalization
- input/output
- error processing
- fault tolerance
- architectural feasibility
- approach to overengineering
- buy vs. build decisions
- change strategy

All decisions should be expressed in a clear language, with considered *alternatives* and *reasons* for the final choice.
